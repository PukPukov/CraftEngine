Для удобства, можно ввести единый стиль кода для проекта:
- Геттеры пишутся в fluent стиле, сеттеры в более подходящем по смысле (если это просто установка данных, то, возможно, сработает флюент стиль, если предполагается что изменение чуть больше чем обновление поля то логичнее выглядит с глаголом set или update (99% случаев) )
- Пустые строки должны сохранять отступы
- Пустая строка на конце файла необходима только для языков в которых эта пустая строка может выполнить какую-либо функцию (для джавы она бесполезна)
- @Override обязательна на методе если может быть применена
- При обращении к полям и методам инстанса надо использовать this
- Все приватные методы должны идти после публичных если они прямо не относятся к какому-то публичному методу, в таком случае класс должен быть подразделён на секции с помощью комментариев

Для удобства, можно ввести единый стиль кода для проекта:
- Пустые строки должны сохранять отступы
- Пустая строка на конце файла необходима только для языков в которых эта пустая строка может выполнить какую-либо функцию (для джавы она бесполезна)
- @Override обязательна на методе если может быть применена
- При обращении к полям и методам инстанса надо использовать this
- Порядок включений в классах такой: константы, состояние, оверрайды, публичные методы, приватные методы. Если какое-то включение принадлежит другому (например, приватное поле используется только в одном методе), то оно может нарушить этот порядок, но эта принадлежность должна быть явно видна, например с помощью комментариев или отсутствия пустой строки между этим полем и методом.

И следующие конвенции (по мере возможности их соблюдения):
- Если значение не Optional и не @Nullable, то оно не может быть null никогда. Классы, соблюдающие это нужно помечать как @NullMarked.
